# Capstone: проектирование аналитического движка

На протяжении курса вы изучали DataFusion послойно: от Arrow-формата ([модуль 1](/course/01-arrow-foundation/01-what-is-apache-arrow)) через архитектуру движка ([модуль 2](/course/02-architecture/01-high-level-overview)) до распределённого выполнения ([модуль 8](/course/08-distributed-execution/01-distributed-computing-basics)). В этом модуле вы соберёте знания воедино, спроектировав и реализовав мини аналитический движок.

## Задание

Построить аналитическую систему на DataFusion Python, которая:

1. **Загружает данные** из Parquet-файлов (arrow-формат на диске --- [модуль 1](/course/01-arrow-foundation/03-parquet-format))
2. **Регистрирует пользовательские функции** (UDF/UDAF --- [модуль 4](/course/04-datafusion-python/04-udf-and-udaf))
3. **Выполняет SQL-запросы** и DataFrame-операции ([модуль 3](/course/03-sql-dataframe-api/01-sql-basics) и [модуль 4](/course/04-datafusion-python/02-dataframe-in-python))
4. **Генерирует отчёты** в Parquet-формате
5. **Использует расширяемость** DataFusion (кастомные источники --- [модуль 5](/course/05-extensibility/01-extension-points))

## Контекст: почему DataFusion Python

В [модуле 4](/course/04-datafusion-python/01-installation-and-context) вы освоили Python-биндинги DataFusion. Для capstone-проекта Python --- оптимальный выбор:

- **Быстрый прототип**: Python позволяет итерировать без компиляции Rust
- **Тот же движок**: Python API вызывает тот же Rust-код через PyO3 --- производительность на уровне нативного DataFusion
- **Готовая экосистема**: PyArrow, Pandas, Parquet --- всё доступно из pip

<Callout type="note">
Capstone-проект не требует знания Rust. Все упражнения выполняются на DataFusion Python, который вы изучали в модуле 4.
</Callout>

Движок состоит из четырёх слоёв:

1. **DataLoader** --- регистрирует файлы данных как таблицы в SessionContext. Аналог каталога в полноценных системах ([модуль 5](/course/05-extensibility/02-custom-table-providers) --- кастомные TableProvider).

2. **UDF Registry** --- регистрирует бизнес-функции. В промышленных системах это часть плагин-архитектуры ([модуль 5](/course/05-extensibility/04-custom-functions) --- кастомные функции).

3. **Query Layer** --- SQL и DataFrame API для аналитических запросов. DataFusion транслирует оба интерфейса в один LogicalPlan ([модуль 2](/course/02-architecture/03-logical-plan)).

4. **Report Generator** --- сохраняет результаты в Parquet. В production-системах этот слой отправляет данные в объектное хранилище или Flight-сервис ([модуль 4](/course/04-datafusion-python/06-advanced-patterns)).

## Самопроверка перед лабораторной

Перед тем как перейти к практике, убедитесь, что вы помните ключевые концепции:

- **SessionContext** --- точка входа в DataFusion Python. Создаётся один раз, используется для регистрации данных и выполнения запросов. Подробно --- [урок 4.1](/course/04-datafusion-python/01-installation-and-context).

- **register_parquet** --- регистрирует файл как таблицу. После регистрации файл доступен по имени в SQL-запросах. Подробно --- [урок 4.1](/course/04-datafusion-python/01-installation-and-context).

- **udf()** --- создаёт скалярную пользовательскую функцию. Принимает Python-функцию, типы аргументов, тип результата и volatility. Подробно --- [урок 4.4](/course/04-datafusion-python/04-udf-and-udaf).

- **Accumulator** --- базовый класс для UDAF. Реализует `state()`, `update()`, `merge()`, `evaluate()`. Подробно --- [урок 4.4](/course/04-datafusion-python/04-udf-and-udaf).

- **write_parquet** --- сохраняет DataFrame в Parquet-файл. Подробно --- [урок 4.2](/course/04-datafusion-python/02-dataframe-in-python).
